# 嵌入式智能大棚蔬菜集群管理系统项目设计报告

## 一、项目背景

传统蔬菜大棚主要依赖人工巡检和经验控制，存在环境变化响应慢、管理成本高、数据难以积累等问题。随着物联网和嵌入式技术的发展，可以通过传感器、微控制器和网络服务构建智能大棚系统，实现环境数据实时采集、设备自动控制和多棚集群管理。

## 二、项目目标

本项目使用 C++ 设计一个嵌入式智能大棚蔬菜集群管理系统，实现以下目标：

1. 采集大棚内温度、空气湿度、土壤湿度、光照和 CO2 数据。
2. 根据环境阈值自动控制风机、水泵、补光灯和通风窗。
3. 支持多个大棚节点集中管理。
4. 提供网页看板展示实时状态和异常预警。
5. 在没有真实硬件时，可以通过模拟服务完成演示。

## 三、系统组成

### 1. 嵌入式节点

每个大棚部署一个 ESP32 节点，负责读取传感器数据，并通过继电器或 PWM 控制执行器。ESP32 具备 WiFi 通信能力，适合农业物联网场景。

### 2. 传感器模块

- DHT22：采集空气温湿度。
- 电容式土壤湿度传感器：采集土壤水分。
- BH1750：采集光照强度。
- MH-Z19B：采集 CO2 浓度。

### 3. 执行器模块

- 风机：用于通风降温。
- 水泵：用于自动灌溉。
- LED 补光灯：用于弱光环境补光。
- 舵机或电机：用于控制通风窗或卷帘。

### 4. 服务端与看板

C++ 集群控制程序负责汇总节点数据、判断预警和生成控制命令。可选网页看板负责展示集群状态，便于管理人员查看和操作。

## 四、控制策略

系统采用基础阈值控制策略：

| 环境条件 | 控制动作 |
| --- | --- |
| 温度高于 28℃ | 开启风机 |
| 温度高于 30℃ | 通风窗开度提高到 85% |
| 土壤湿度低于 45% | 开启水泵 |
| 光照低于 12000 lux | 开启补光灯 |
| CO2 高于 1000 ppm | 开启风机通风 |

该策略结构简单，便于在嵌入式设备上运行，也方便后期升级为 PID 控制或模型预测控制。

## 五、项目创新点

1. 将单个大棚控制扩展为多大棚集群管理。
2. 使用 C++ 类封装节点、传感器、执行器和控制器，结构清晰。
3. 既有嵌入式固件，又有上位机服务和网页展示。
4. 没有硬件时仍可通过模拟数据完整演示。
5. 系统结构清晰，便于后续加入图像识别、天气预测和数据库存储。

## 六、运行效果

运行 C++ 程序后，命令行会显示 6 个大棚节点的实时状态，包括温度、空气湿度、土壤湿度、光照、CO2 和执行器状态。当环境异常时，系统会输出高温、缺水、光照不足等提示。可选网页服务可进一步展示可视化看板。

## 七、后续改进

1. 接入真实 MQTT 服务器，实现硬件节点和服务端通信。
2. 增加 SQLite 或 InfluxDB，保存历史数据。
3. 增加手机端或微信小程序。
4. 引入摄像头识别病虫害和作物生长状态。
5. 根据天气预报优化通风、补光和灌溉策略。
