# 系统架构

## 1. 总体设计

系统采用“感知层、控制层、服务层、展示层”四层结构。

- 感知层：温湿度、土壤湿度、光照、CO2、摄像头等传感器。
- 控制层：ESP32 节点、继电器、水泵、风机、补光灯、通风窗。
- 服务层：本地 Python 服务或云端服务，负责数据汇总、预警判断和控制指令下发。
- 展示层：网页看板或移动端，用于查看状态和手动控制。

## 2. 数据流

1. ESP32 定时采集传感器数据。
2. 节点通过 MQTT 或 HTTP 上报数据。
3. 服务端保存数据并判断是否异常。
4. 看板拉取接口显示实时状态。
5. 用户手动控制或自动策略生成控制命令。
6. 命令下发到节点，节点驱动执行器动作。

## 3. 自动控制策略

| 条件 | 动作 |
| --- | --- |
| 温度高于 28℃ | 开启风机 |
| 温度高于 30℃ | 通风窗开到 85% |
| 土壤湿度低于 45% | 开启水泵 |
| 光照低于 12000 lux | 开启补光灯 |
| CO2 高于 1000 ppm | 开启风机通风 |

## 4. 集群管理

每个大棚节点都有唯一编号，例如 `GH-01`、`GH-02`。服务端按节点编号统一管理状态，适合扩展到多个蔬菜区域。
